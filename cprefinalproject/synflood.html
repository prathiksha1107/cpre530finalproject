SYN Flood Attack

What is a SYN Flood Attack?

SYN Flood attack in the Transport Layer can be compared to an overwhelmed restaurant reservation system:

Imagine a popular restaurant that operates on a reservation-only basis. Customers call the restaurant to book a table, and each reservation request is answered by a host. The host notes down the customer's name and assigns a table, expecting the customer to arrive at a specified time. This process is generally smooth and efficient.

However, one day, a group decides to disrupt the restaurant's operation. They start making numerous fake reservation calls, each time providing a different name and then not showing up. The host, following protocol, allocates tables for these fake reservations. As a result, the restaurant appears to be fully booked, with many tables reserved and kept empty, awaiting guests that will never arrive.

In this scenario:

    • The restaurant's reservation system represents the Transport Layer of a network, particularly the process that handles incoming connection requests (SYN requests in TCP/IP).
    • The host is akin to the server managing incoming SYN requests.
    • Each fake reservation call symbolizes a malicious SYN packet sent during a SYN Flood attack.
    • The unavailability of tables for actual customers due to fake reservations is similar to legitimate users being unable to establish connections because the server is overwhelmed with false requests.
    • Guests occupying aisles and emergency exits symbolize the overflow of data into adjacent memory areas.

Just as the restaurant's reservation system is overwhelmed and cannot accommodate real customers, a SYN Flood attack inundates a server with a flood of SYN requests, leaving it unable to process legitimate requests. This disrupts normal operations and can lead to a denial of service for actual users trying to connect to the server.
